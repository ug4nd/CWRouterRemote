# CFRRemote simple tunnel + v2rayA web UI

## Новая логика

Программа делает только подготовку удалённого доступа:

- устанавливает `cloudflared`;
- устанавливает `luci-app-cloudflared`;
- настраивает и запускает Cloudflare Tunnel по token;
- устанавливает `v2raya`;
- устанавливает `luci-app-v2raya`;
- запускает сервис v2rayA, чтобы был доступен web UI на `127.0.0.1:2017`.

Программа больше НЕ делает:

- не сохраняет VLESS ключ;
- не импортирует VLESS;
- не включает VPN-режимы;
- не настраивает tproxy/RoutingA автоматически.

VLESS и VPN настраиваются вручную через v2rayA web UI по Cloudflare route.

## Cloudflare routes

Пример:

```text
ugandnetis1-luci.ug4nd.ru   -> http://127.0.0.1:80
ugandnetis1-v2raya.ug4nd.ru -> http://127.0.0.1:2017
```

## Запуск

```bash
pip install -r requirements.txt
python src/main.py
```


## Проверка внешнего IP

Добавлена кнопка `Проверить внешний IP`.

Она подключается к роутеру по SSH и выполняет запрос с самого роутера:

```text
https://api.ipify.org
```

Так можно проверить, через какой IP роутер выходит в интернет после ручной настройки VPN в v2rayA.


## GUI groups + IP check box

- Галочки разделены на группы: `Общее`, `cloudflared`, `v2rayA`.
- Внешний IP больше не находится в окне Cloudflare Tunnel.
- Добавлено отдельное окно: `Проверить IP адрес роутера через VPN`.
- В этом окне есть поле `Адрес сервера`, кнопка `Проверить IP` и поле результата.


## Router IP field update

Окно проверки IP упрощено:

- убрано поле `Адрес сервера`;
- оставлена кнопка `Проверить IP`;
- добавлено поле `IP адрес роутера`, куда выводится внешний IP после проверки;
- сервис проверки скрыт в коде/JSON по умолчанию: `https://api.ipify.org`.

Смысл: пользователь нажимает `Проверить IP`, видит внешний IP роутера и сравнивает его с IP своего VPN-сервера.
